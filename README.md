# Constellation — An Atlas of Art History

An interactive star-map of art history and a walkable 3D gallery for every artist.
Zoom through eight centuries of painting as a constellation of movements, open a
museum-placard card for any master, then step inside their gallery and inspect
each work up close.

Every biography, date, painting, story and fact is drawn from **Wikipedia and
Wikimedia Commons**. Nothing is generated by AI — if it isn't on Wikipedia, it
isn't here.

## Features

- **The Constellation** — a zoomable, infinite-canvas timeline where each art
  movement is a glowing galaxy placed by date. Zoom into one and its artists
  rise as stars linked by influence lines.
- **Explore filter** — a dropdown that swaps between Movements and Artists with
  GSAP animation and live search; pick one to fly there and dim the rest.
- **Artist cards** — a timeless museum placard with portrait, dates, nationality
  and a short biography, all from Wikipedia.
- **Walkable 3D museums** — a first-person gallery (drag to look, WASD to walk)
  hung with the artist's real paintings under individual spotlights, with a
  reflective floor, soft shadows, ACES tone mapping and bloom.
- **Inspect** — click any painting for a high-resolution lightbox with its title,
  year, the story behind it and factual details.

## Scope

12 movements from Medieval & Gothic through Expressionism are covered in depth
(3+ artists each, 8+ paintings per gallery). Later movements (Cubism →
Contemporary) appear for the full sweep of the timeline, but their artists are
still in copyright — Wikimedia Commons can't host those paintings freely — so
those galleries are sparse or card-only by design.

## Stack

- Next.js 16 (App Router) · React 19 · TypeScript · Tailwind v4
- react-three-fiber · drei · @react-three/postprocessing (the galleries)
- GSAP (the filter)
- Neon Postgres · Drizzle ORM
- Data: Wikidata SPARQL · Wikipedia REST · Wikimedia Commons (`Special:FilePath`)

## Local development

```bash
cp .env.example .env.local      # add your Neon DATABASE_URL
npm install
npm run db:push                 # create the tables
npm run ingest                  # populate from Wikipedia/Wikidata/Commons (~5 min)
npm run dev                     # http://localhost:3000
```

`npm run ingest:dry` reports per-artist painting counts without writing.

## Deploy

Deploys on Vercel. Set `DATABASE_URL` (your Neon connection string) in the
project's environment variables — it is never committed.
